# コードレビューワークフロー

変更されたコードを対象に包括的なレビューを実施する。

---

## Step 1: 変更の把握

```bash
git diff HEAD
git status
```

変更ファイルの一覧と差分を確認する。

---

## Step 2: レビュー実施

以下の観点でレビューする：

### 機能性
- [ ] 仕様（SPEC.md）通りに実装されているか
- [ ] エッジケースが処理されているか
- [ ] エラーハンドリングが適切か

### コード品質
- [ ] 命名規則（CLAUDE.md のコーディング規約）に従っているか
- [ ] 不要な複雑さ・過度な抽象化がないか
- [ ] コードの重複がないか

### セキュリティ
- [ ] SQLインジェクション・XSS等のOWASP Top 10 脆弱性がないか
- [ ] 機密情報がハードコードされていないか
- [ ] 入力バリデーションが適切か

### テスト
- [ ] 新機能にテストが書かれているか
- [ ] 既存テストが通るか

---

